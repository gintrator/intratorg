Beaker

Source

Put it in a bottle and shake it all up, affectionately named after Flask.

This project is an experiment in building a small full-stack web service. This is mainly an attempt to figure out how something like Flask is implemented by duplicating some of its features. Beaker implements the WSGI app/server interface, which allows it to run on any WSGI compliant Python server. This project also contains a simple implementation of a WSGI server and an example app.

Endpoints are declared via decorators, as in Flask. You can also declare the type of content that the endpoint will return. URL parameters are forwarded into the handling function in req.args, and path variables are injected as named variables into the function. Here's an example. 

app = Beaker('Sample App')

@app.get('/person/<name>/<age>', mimetype='text/html')
def person(req, name, age):
html = "<p>Hello my name is {0} and I am {1} years old.<p>".format(name, age)
return Response(body=html, status=200)
A request to the endpoint '/person/bob/21' will return the HTML '<p>Hello my name is bob and I am 21 years old.<p>'.
